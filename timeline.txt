Great question. It is easy to think of a `.tif` (TIFF) file as just a high-quality photograph, but in Big Data and Data Science (specifically GIS/mapping), it is actually a **GeoTIFF**.

Think of the `pak_ppp_2020.tif` file you downloaded as a massive, invisible Excel grid laid over the entire map of Pakistan.

Here is exactly what is inside it:

**1. The Grid (Pixels)**
Instead of an image made of colored pixels, the map of Pakistan is divided into millions of tiny squares. Because you downloaded the "100m resolution" 
version, every single "pixel" in this file represents a real-world area of **100 meters by 100 meters** on the ground.

**2. The Data (Pixel Values)**
In a normal photo, a pixel stores a color (like "Blue"). In a GeoTIFF, the pixel stores a **number**. For this specific WorldPop dataset, 
that number is the **estimated number of people living inside that 100m square**.

* A pixel over the Cholistan Desert might have a value of `0` or `0.05`.
* A pixel over a dense neighborhood in Karachi might have a value of `850`.

**3. The Coordinates (Spatial Metadata)**
The file also contains hidden metadata that tells the computer exactly where on Earth each pixel belongs using Latitude and Longitude.

### **Why we are using it for Big Data:**

If we extract the data from this image, we aren't looking at a picture. We are turning it into a table with three columns: **Latitude**, **Longitude**, and **Population**.

Because Pakistan is huge and 100 meters is very small, this file contains **millions** of pixels. When you flatten it out into a table in Task 3, it will generate millions 
of rows of data—which is exactly why your teacher wants you to use Apache Spark instead of standard Python!

Task 3 Completion Report: Spatial Data Cleaning & Preprocessing
1. What We Did (The Objective & Execution)
In this phase, we extracted, cleaned, and transformed raw spatial raster data into a distributed tabular format suitable for Machine Learning.

Data Extraction: Used rasterio to read the pak_ppp_2020.tif file, filtering out empty pixels (population < 0.1) to extract 76.7 million populated spatial coordinates.

Optimization: Downsampled the dataset to 3 million coordinates to prevent local RAM overflow while maintaining a statistically significant "Big Data" volume.

PySpark Transformation: Converted the arrays into a distributed PySpark DataFrame. Applied bounding box filters to strictly confine Latitude (23.0 to 37.0) and Longitude (60.0 to 77.0) to Pakistan's borders. Rounded coordinates and cast population values to uniform data types.

Data Storage: Exported the final cleaned DataFrame into the highly compressed Parquet format.

2. Why We Did It (The Rationale)

Format Conversion: Machine Learning algorithms cannot natively read .tif images. We had to flatten the spatial grid into a tabular format (Longitude, Latitude, Population).

Memory Management: 76 million rows will crash most local laptops. Downsampling to 3 million rows preserves the geographic distribution patterns while allowing your local Spark cluster to process it efficiently.

Parquet Standardization: Saving the output as a .parquet file replaces the slow, uncompressed CSV format with a columnar storage format optimized for Big Data query performance.


import os
import glob
import rasterio
import numpy as np
import pandas as pd
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, lit, round

# 1. Java 11 & Hadoop Dynamic Pathing Fix
project_root = os.getcwd()
java11_dir = os.path.join(project_root, "java11_folder")
for item in os.listdir(java11_dir):
    if "jdk" in item.lower():
        os.environ['JAVA_HOME'] = os.path.join(java11_dir, item)
        break
os.environ['HADOOP_HOME'] = r'C:\hadoop'

# 2. Initialize Spark with extra memory
spark = SparkSession.builder \
    .appName("DataCleaning") \
    .config("spark.driver.memory", "4g") \
    .getOrCreate()
spark.sparkContext.setLogLevel("ERROR")

# 3. Process the WorldPop TIF file
wp_path = glob.glob(os.path.join(project_root, "data", "raw", "worldpop", "*.tif"))
with rasterio.open(wp_path[0]) as src:
    data = src.read(1)
    mask = data > 0.1 
    rows, cols = np.where(mask)
    xs, ys = rasterio.transform.xy(src.transform, rows, cols)
    pops = data[rows, cols]

# Downsampling to 3 million points
if len(pops) > 3000000:
    sample_indices = np.random.choice(len(pops), 3000000, replace=False)
    xs = np.array(xs)[sample_indices]
    ys = np.array(ys)[sample_indices]
    pops = pops[sample_indices]

# 4. Push to Spark
pdf = pd.DataFrame({"lon": xs, "lat": ys, "population": pops})
df = spark.createDataFrame(pdf)

# 5. Clean and Transform in PySpark
df_clean = df.filter(
    (col("lat") >= 23.0) & (col("lat") <= 37.0) &  
    (col("lon") >= 60.0) & (col("lon") <= 77.0)    
).dropna()

df_clean = df_clean.withColumn("lat", round(col("lat"), 5)) \
                   .withColumn("lon", round(col("lon"), 5)) \
                   .withColumn("population", col("population").cast("double")) \
                   .withColumn("year", lit(2020))

# 6. Save to Parquet
out_dir = os.path.join(project_root, "data", "processed", "cleaned.parquet")
df_clean.write.mode("overwrite").parquet(out_dir)

spark.stop()

import os
from pyspark.sql import SparkSession
from pyspark.sql.functions import col, log1p, when, concat_ws, round

# 1. Java 11 & Hadoop Dynamic Pathing Fix
project_root = os.getcwd()
java11_dir = os.path.join(project_root, "java11_folder")
for item in os.listdir(java11_dir):
    if "jdk" in item.lower():
        os.environ['JAVA_HOME'] = os.path.join(java11_dir, item)
        break
os.environ['HADOOP_HOME'] = r'C:\hadoop'

# 2. Initialize Spark
spark = SparkSession.builder \
    .appName("FeatureEngineering") \
    .config("spark.driver.memory", "4g") \
    .getOrCreate()
spark.sparkContext.setLogLevel("ERROR")

# 3. Load the cleaned Parquet data
input_path = os.path.join(project_root, "data", "processed", "cleaned.parquet")
df = spark.read.parquet(input_path)

# 4. Feature Engineering
# Feature A: Grid ID (Grouping into 1km Macro-regions by rounding coordinates)
df_feat = df.withColumn("grid_id", concat_ws("_", round(col("lat"), 2), round(col("lon"), 2)))

# Feature B: Urbanization Label (1 = Urban, 0 = Rural)
df_feat = df_feat.withColumn("is_urban", when(col("population") > 50.0, 1).otherwise(0))

# Feature C: Density Score (Log transformation for ML stability)
df_feat = df_feat.withColumn("density_score", log1p(col("population")))

# 5. Save ML-Ready Data
output_path = os.path.join(project_root, "data", "processed", "ml_features.parquet")
df_feat.write.mode("overwrite").parquet(output_path)

# 6. Data Summary
total_count = df_feat.count()
urban_count = df_feat.filter(col("is_urban") == 1).count()
rural_count = total_count - urban_count

print("\n--- Sprawl Statistics ---")
print(f"Total Pixels: {total_count:,}")
print(f"Urban Pixels: {urban_count:,} ({(urban_count/total_count)*100:.2f}%)")
print(f"Rural Pixels: {rural_count:,} ({(rural_count/total_count)*100:.2f}%)")

spark.stop()
